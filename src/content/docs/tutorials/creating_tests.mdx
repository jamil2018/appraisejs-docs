## What you'll build

<OutcomeCard title="A reusable smoke scenario" tone="success">
  You will create one module, one environment, one locator group, four locators,
  one suite, and one test case that can be rerun from the Test Runs screen.
</OutcomeCard>

## Step-by-step procedure

<UIPath
  steps={["Modules", "Environments", "Locators", "Test Suites", "Test Cases"]}
/>

1. Create module `Authentication` so the scenario has a clear home in the product model.
2. Create environment `Local` with the application base URL you want to test.
3. Open `Locators -> Create` and launch the Chromium picker from that saved environment.
4. Capture locators for `emailInput`, `passwordInput`, `submitButton`, and `welcomeBanner`.
5. Accept the suggested locator group for the login route, or create `Login Page` if nothing relevant exists yet.
6. Create suite `Auth Smoke` under the `Authentication` module.
7. Create case `Valid login shows welcome banner` and attach it to the `Auth Smoke` suite.
8. Add the first step to navigate to the selected environment base URL so the case is portable across environments.
9. Add input steps for email and password using the captured locators.
10. Add a click step for the submit button.
11. Add an assertion step that checks the `welcomeBanner` visibility is `true`.
12. Save the case and re-open it once to confirm every parameter, locator reference, and suite relationship still looks correct.

## Checkpoint

- The case opens with a complete ordered flow and no missing required parameters.
- Every referenced locator exists and belongs to a route-aware locator group.
- The suite, module, and environment names are clear enough that another team member could reuse them later.

## Common mistakes

- Creating locators before deciding which environment or route they should belong to.
- Saving a case with literal values that should have been environment-driven or reusable.
- Mixing multiple business outcomes into one case instead of one clear scenario.
- Naming assets too generically, which makes future reuse harder.
