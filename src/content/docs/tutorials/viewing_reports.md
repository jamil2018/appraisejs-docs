This tutorial covers practical triage using report pages.

## Workflow

1. Open `Reports` list and sort by newest failed runs.
2. Open a failed report.
3. Review total pass/fail metrics and environment metadata.
4. Drill into failed scenario rows.
5. Cross-reference run logs and trace artifacts.
6. Fix test or app issue, then re-run targeted scope.

## Fast triage checklist

- Did locator resolution fail?
- Did route/assertion mismatch occur?
- Did environment config drift?
- Is failure flaky across browser engines?
