New scaffolds start with a seeded local database and a clean automation
workspace. That means the steps below create your first real environment,
locator group, suite, and case instead of modifying bundled starter features or
reports.

## Step-by-step procedure

<UIPath
  steps={[

    "Test Runs",
    "Reports",

1. Start AppraiseJS with `npm run start` and confirm the local app opens in your browser. Use `npm run dev` only if you intentionally need the dev-server workflow while iterating locally.
2. Create environment `Local` with the base URL of the application you want to test.
3. Create module `Authentication` or another feature-oriented module name.
4. Open `Locators -> Create` and launch the Chromium picker from the saved environment.
5. Capture at least these locators: `emailInput`, `passwordInput`, `submitButton`, and one success-state locator such as `welcomeBanner`.
6. Accept the suggested locator group if the route is already recognized, or create a new group such as `Login Page`.
7. Create test suite `Auth Smoke`.
8. Create test case `Valid login shows welcome banner`.
9. Add these steps in order: navigate to selected environment base URL, fill email input, fill password input, click submit button, assert success-state visibility.
10. Save the test case and open `Test Runs -> Create`.
11. Create a run against `Local`, submit it, and watch status changes on the run details page.
12. Open the linked report and confirm the scenario outcome matches what happened in the browser.

## Checkpoint

- You can open one environment, one module, one locator group, one suite, and one case from the UI.
- Your test run either completed successfully or failed with a report and logs you can inspect.
- You know whether you need to continue to [Creating Tests](/tutorials/creating_tests/) or jump to [Troubleshooting](/operations/troubleshooting/).

## Common mistakes

- Expecting the scaffold to include starter environments, locators, test cases, or reports.
- Launching the picker from the wrong base URL and saving locators for the wrong page.
- Creating a run before the test case has all required step parameters.
- Forgetting that AppraiseJS uses the selected environment during navigation and wait steps.
- Skipping `Settings -> Sync` after editing generated files outside the UI.
