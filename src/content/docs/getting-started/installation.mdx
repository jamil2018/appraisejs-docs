## Step-by-step procedure

### Recommended path: scaffold with `create-appraisejs`

```bash
npx create-appraisejs@latest
cd <your-project-name>
npm run setup # only if you skipped setup during scaffolding
npm run start
```

The examples on this page use `npm`, but the scaffold rewrites scripts and
printed next commands for the package manager you selected.

`create-appraisejs` is the official AppraiseJS bootstrapper. The default
experience uses the bundled template shipped with the package, not a repo
download step.

### What the scaffold does

- Prompts for the target directory.
- Prompts for the package manager: `npm`, `pnpm`, `yarn`, or `bun`.
- Prompts for whether to run production setup immediately.
- Prompts for which Playwright browsers to install.
- Uses the bundled template by default.
- Rewrites generated project scripts to match the selected package manager.
- Preserves the seeded SQLite database at `prisma/dev.db`.
- Resets the automation workspace so `automation/config/environments/environments.json` starts as `{}` and `automation/mapping/locator-map.json` starts as `[]`.
- Includes reusable step definitions, but does not bundle starter features, locators, or reports.
- Runs the project `setup` script and installs the selected Playwright browsers if you opt into setup during scaffolding.
- Prints the exact next commands to run if you skip setup during scaffolding.

After startup, open the local URL shown by the app. `npm run dev` is still
available for local iteration, but it is not the primary onboarding path.

## Checkpoint

- The local app starts without blocking setup errors.
- The AppraiseJS UI loads in the browser.
- The project still contains the seeded `prisma/dev.db` database.
- The automation workspace is clean rather than preloaded with starter artifacts.
- You know whether to continue to [Quick Start](/getting-started/quick_start/) or jump to [Troubleshooting](/operations/troubleshooting/).

## Common mistakes

- Skipping `npm run setup` and then starting the app without running the printed next commands.
- Assuming the scaffold downloads a remote repo instead of using the bundled template.
- Expecting starter features, locators, or reports to already exist in the generated project.
- Treating browser dependency errors as AppraiseJS UI bugs instead of installation issues.
- Continuing into test authoring before confirming the app actually loads locally.

## Recovery steps

- Migration errors: run `npm run migrate-db`.
- Browser dependency errors: run `npm run install-playwright`.
- Stale or missing generated entities after setup: use `Settings -> Sync` after the UI loads, or run `npm run sync-all` if the UI is unavailable.
